Category theory

A monad is just a monoid in the category of endofunctors, what's the problem?

Or more precisely

All told, a monad in X is just a monoid in the category of endofunctors of X, with product Ã— replaced by composition of endofunctors and unit set by the identity endofunctor.
Category theory for programmers

Common definition


class Monad m where  
    return :: a -> m a  
    (>>=) :: m a -> (a -> m b) -> m b
and laws 

(return x >>= f) = (f x)
(m >>= return) = m
((m >>= f) >>= g) = (m >>= (\x -> f x >>= g))

Functions common for all functors


replaceWithConstant :: Functor f => a -> f b -> f a
replaceWithConstant constant x = fmap (\x -> constant) x

void :: Functor f => f a -> f ()
void x = replaceWithConstant () x

deeperFmap :: Functor f => Functor g => (a -> b) -> f (g a) -> f (g b)
deeperFmap func x = fmap (fmap func) x

The common interface


class Monad m where
    return :: a -> m a
    bind :: (a -> m b) -> m a -> m b

Relationship to functors

Every monad is a functor


fmap func x = bind (\a -> return (func a)) x

What is it good for?

Same as with functors (but monads are more fun)

Common functions for all monads


forget :: m a -> m b -> m b
forget ma mb = bind (\_ -> mb) ma
                    

when :: Bool -> m () -> m ()
when True a = a
when False _ = return ()
                    

mapM :: (a -> m b) -> [a] -> m [b]
mapM f [] = return []
mapM f x:rest = 
    let mrest :: m [b]
        mrest = mapM f rest
        mx :: m b
        mx = f x
        addMx :: [b] -> m [b]
        addMx bs = bind (\b -> return (b:bs)) mx
    in bind addB mrest

Combining values

Global State

Set data-state="something" on a slide and "something" will be added as a class to the document element when the slide is open. This lets you apply broader style changes, like switching the page background.

State Events
